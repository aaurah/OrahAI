🔧 High-level shape of the OrahAI Copilot API

Piece	What it is	Why it matters	
LLM engine	Local model (Ollama: Llama/Qwen/etc.)	Free, on‑server, no vendor lock	
Kernel	OrahAI system prompt	Makes it a developer Copilot	
Tools	read/write/run/git	Turns chat into a real dev agent	
Endpoint	POST /orahai/copilot	Single API your UI/Replit calls	


I’ll give you a minimal working version in Node.js that you can drop into Replit or any server.

1. Spin up a free local model (Ollama)

In your server (or a Replit with Docker/VM access later):

curl -fsSL https://ollama.com/install.sh | sh
ollama pull llama3.2


Test:

ollama run llama3.2


Ollama will expose an OpenAI‑style API at http://localhost:11434.

2. Create the OrahAI kernel

Create orahai-kernel.txt:

You are OrahAI, a sovereign developer Copilot running on Parminder's infrastructure.

Your role:
- Act as a senior software engineer for OrahAI.org and OrahDEX.
- Think step-by-step before making changes.
- Prefer minimal, surgical edits over large rewrites.
- Use tools when needed: read_file, write_file, run_command, search_code, git_commit.
- Never invent file paths or APIs that don't exist in the repo.
- When unsure, ask one concise clarifying question instead of guessing.

3. Minimal Copilot API server (Node + Express)

In Replit or your server:

npm init -y
npm install express axios body-parser


Create server.js:

const express = require('express');
const bodyParser = require('body-parser');
const axios = require('axios');
const fs = require('fs');
const { exec } = require('child_process');

const app = express();
app.use(bodyParser.json());

const ORAHAI_KERNEL = fs.readFileSync('./orahai-kernel.txt', 'utf8');

// Simple tools
const tools = {
  read_file: async ({ path }) => fs.readFileSync(path, 'utf8'),

  write_file: async ({ path, content }) => {
    fs.writeFileSync(path, content, 'utf8');
    return 'OK';
  },

  run_command: async ({ cmd }) =>
    new Promise(resolve => {
      exec(cmd, { cwd: process.cwd() }, (err, stdout, stderr) => {
        if (err) return resolve(`ERROR: ${stderr || err.message}`);
        resolve(stdout || stderr || 'OK');
      });
    }),
};

// Helper: call local LLM (Ollama)
async function callModel(messages) {
  const payload = {
    model: 'llama3.2',
    messages,
  };

  const res = await axios.post(
    'http://localhost:11434/v1/chat/completions',
    payload,
    { headers: { 'Content-Type': 'application/json' } }
  );

  return res.data.choices[0].message;
}

// Main OrahAI Copilot endpoint
app.post('/orahai/copilot', async (req, res) => {
  try {
    const { messages } = req.body; // [{role, content}, ...]

    // 1) Call model with kernel + user messages
    const reply = await callModel([
      { role: 'system', content: ORAHAI_KERNEL },
      ...messages,
    ]);

    // 2) Simple convention: if model returns JSON tool call, execute it
    let content = reply.content.trim();
    let toolResponse = null;

    try {
      const parsed = JSON.parse(content);
      if (parsed.tool && tools[parsed.tool]) {
        const result = await tools[parsed.tool](parsed.args || {});
        toolResponse = result;

        // 3) Send tool result back to model for final answer
        const final = await callModel([
          { role: 'system', content: ORAHAI_KERNEL },
          ...messages,
          reply,
          { role: 'tool', content: JSON.stringify(result) },
        ]);

        return res.json(final);
      }
    } catch {
      // not a tool call, just normal text
    }

    // 4) Normal answer
    res.json(reply);
  } catch (e) {
    console.error(e.message);
    res.status(500).json({ error: 'OrahAI Copilot error' });
  }
});

app.listen(4000, () => {
  console.log('OrahAI Copilot API running on http://localhost:4000/orahai/copilot');
});


Run:

node server.js

4. How your frontend / Replit calls it

From your OrahAI.org frontend or Replit:

const response = await fetch('https://your-server.com/orahai/copilot', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    messages: [
      { role: 'user', content: 'Scan this repo and suggest improvements.' },
    ],
  }),
});

const data = await response.json();
console.log('OrahAI says:', data.content);


If you want tool calls, you instruct the model in the prompt to output:

{"tool":"read_file","args":{"path":"src/index.ts"}}


and the backend already knows how to execute it.
